    # vue2 视图不更新

    # vue2响应式核心

    使用 Object.defineProperty,为新对象或已存在对象添加属性,以及针对这个属性的 get 和 set 操作

    let obj = {
  name: '哈哈',
  age: 2
}
let obj2 = {}
Object.keys(obj).forEach(key => {
  Object.defineProperty(obj2, key, {
    get() {
      console.log(`get ${key}`)
      return obj[key]
    },
    set(newVal) {
      console.log(`set ${key}:${newVal}`)
      obj[key] = newVal
    }
  })
})
console.log(obj2.name)
obj2.age = 100
console.log(obj2.age)
obj2.sex = 1
console.log("sex:", obj2.sex)
/*
output:
get name
哈哈
set age:100
get age
100
sex: 1
*/

    对象新增属性不更新页面的问题出现了 给 obj2 新增属性,值有效,但并没有执行 set 和 get,这是因为没有对新属性通过Object.defineProperty添加 get/set

    vue 提供的解决办法,

    1. Vue.set(obj2, 'sex', 1) or 组件内部this.$set(obj2,'sex',1)
    2. obj2 = Object.assign({}, obj2, { sex: 1 }) 把原对象,和要添加的数据进行合并,保存回去 Object.assign(obj2, 'sex', 1)这种不会触发更新

    # 数组更新问题

    首先明确的一个点是Object.defineProperty 是支持监听数组的,以下标为 key 进行监听,网上很多说不支持监听的,这样不对

    let arr1 = [100, 200, 300]
let arr2 = []
Object.keys(arr1).forEach(keyIndex => {
  Object.defineProperty(arr2, keyIndex, {
    get() {
      console.log(`get ${keyIndex}`)
      return arr1[keyIndex]
    },
    set(newVal) {
      console.log(`set ${keyIndex}: ${newVal}`)
      arr1[keyIndex] = newVal
    }
  })
});
arr2[2] = 201
console.log(arr2[2])
/*
output:
set 2: 201
get 2
201
*/

    但是在 vue 中,出于性能考虑(数组长度不可控,长列表等),vue底层没有为数组的下标创建响应式 get/set, 而是处理了数组常用的七个操作方法,在这些方法里添加了响应式操作 push, pop, unshift, shift, reverse, sort, splice 使用下标操作数组时,不会走上面的任一方法,也就不会触发页面更新

    解决:

    1. 使用 splice批处理,splice(startIndex, deleteCount, item1, item2, itemN)
    2. 使用 Vue.set() or this.$set, 和上面对象类似
