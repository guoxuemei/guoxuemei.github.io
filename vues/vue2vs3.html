    # Vue2 vs Vue3

    # 1. 响应式实现方式

    vue2 使用 Object.defineProperty,来为组件对象的 data 对象的每个属性添加监听,设置 get 和 set,并且在 set 中添加了通知机制

    1. 对象新增/删除属性,无法触发 set,需要Vue.set方法实现
    2. 基于性能和用户体验的性价比,vue2 没有为数组的下标添加监听, 但是为数组的每个元素又使用了Object.defineProperty, vue2底层处理了常用的七个数组方法,使用这几个方法可以正常处理响应式

    vue3 使用了 es6 中的 Proxy, 把对象类型包装成一个代理对象,针对这个对象的增删改查,都可以被监测到

    • reactive方法里面使用了 new Proxy
    • 针对基本数据类型,提供了 ref 包装,添加了 get value,和 set value,操作 ref 对象要使用.value获取真实值

    # 2.mixin vs composition function

    vue2 的组件,是以对象形式表达,想复用功能,就没办法往对象里对应的属性上添加, 使用mixin字段,动态将 mixin 里的各字段进行添加,合并

    缺点: 数据来源不清晰,有同名冲突的可能

    组合式函数,利用组合式api,封装和复用有状态逻辑的函数

    # 3.v-if 和 v-for

    vue2 中,v-for 优先级较 v-if 高,一起使用会先执行循环,拿到 item,再去判断是否显示该 item,如果要隐藏,会导致浪费,不建议一起使用 可以将数据提前进行一个 filter,然后在 v-for 渲染

    vue3 中 v-if 的优先级更高,和 v-for 一起使用时,v-if 无法使用循环变量,会报错,因为v-if 先执行,v-for 还没执行呢

    可以将 v-for 作用于 template,然后在使用 v-if,不过还是回到 vue2 的性能考虑问题

    <template v-for="todo in todos">
  <li v-if="!todo.isComplete">
    {{ todo.name }}
  </li>
</template>

    # 4. 根元素

    vue3 不要求template单根节点了,大部分情况下多包一层,是种浪费

    但也衍生出新的问题: 父组件通过在子组件身上设置class,style,自定义事件,应该作用于哪个节点呢

    单根节点组件,有个自动透传行为,可以配置是否自动透传inheritAttrs,

    • 为 true 则透传进来的属性会合并到当前节点元素上,外部透传属性优先级要高于组件内部定义的 class 等
    • 为 false 则传入组件的 class,style,自定义事件存储于$attrs上,选择性使用

    多根节点没有自动透传,需要使用v-bind="$attrs"显式绑定到哪个根节点上

    # 5.模板引用

    vue2 直接在元素或组件上设置 ref="someid",然后通过$refs.someid获取,操作dom

    vue3,使用 ref 创建引用对象,绑定到元素上, v-for使用一个 ref([])数组引用

    # 6. 组件上的 v-model

    vue 实现了表单元素的 v-model,我们也可以做组件的 v-model

    核心 就是绑定一个 modelValue,触发一个update:modelValue事件,命名格式要求

    <CustomInput v-model="searchText" />


<!-- CustomInput.vue -->
<script setup>
defineProps(['modelValue'])
defineEmits(['update:modelValue'])
</script>

<template>
  <input
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>

    <MyComponent v-model:title="bookTitle" />
vue
<!-- MyComponent.vue -->

<script setup>
  //默认收集 modelValue, 可以自定义argument值
defineProps(['title'])
defineEmits(['update:title'])
</script>

<template>
  <input
    type="text"
    :value="title"
    @input="$emit('update:title', $event.target.value)"
  />
</template>

    # 7. 组合式api 和 <script setup>

    组合式api 提供了 setup 函数,将属于同一块业务的数据,方法,归集到一起,在这里取代了 beforeCreate,created 钩子

    <script setup> 简化 setup 函数,定义完不用再手动暴露
