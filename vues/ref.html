    # ref/reactive

    # vue3的响应式基础

    因为 vue2 使用的 Object.defineProperty,在面对特定类型时具有缺陷(看这篇)

    vue3 底层使用了 es6 中的 Proxy 来做数据劫持

    
let obj = {
  name:"哈哈",
  age:18
}
let handlers = {
  get(target,key) {
    console.log(`get ${key}`)
    return target[key]
  },
  set(target,key,value) {
    console.log(`set ${key}:${value}`)
    target[key] = value
    return true
  }
}
let proxyObj = new Proxy(obj,handlers)

console.log(proxyObj.age)
proxyObj.name = "嘿嘿"
console.log(proxyObj.name)
proxyObj.height = 1.2
/**
output:
get age
18
set name:嘿嘿
get name
嘿嘿
set height:1.2
 */

    直接给对象新增属性也能监测到了

    vue3的 reactive 函数,底层就是使用了 proxy,返回代理对象

    # reactive 短板

    Proxy 只能代理对象类型,基本数据类型无法进行代理,vue又提供了 ref 方法,用来包装基本数据类型 RefImpl 伪代码

    class RefImpl{
  constructor(target) {
    let isObj = isObject(target)
    if (isObj) {
      this._value = toReactive(target)
    } else {
      this._value = target
    }
  }
  get value() {
    return this._value
  }

  set value(newVal) {
    let isObj = isObject(newValue)
    this._value = isObj ?  toReactive(newVal) : newVal
    // ...触发响应式
  }
}

    使用 ref 会生成一个 ref 对象, 里面包含了 value 的 getter 和 setter,这也是为什么要用.value 解包ref值的原因

    包装对象类型时,底层会调 reactive 方法,将数据值包装为一个 proxy 对象,保存在_value 中

    包装基本数据类型时,不做处理只保存在_value

    # 使用区别

    基本数据类型只能使用 ref 包装

    对象类型可以用 ref,也可以用 reactive, 逻辑代码使用 ref 时,需要使用.value 解包数据
