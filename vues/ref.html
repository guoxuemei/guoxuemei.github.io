    # ref/reactive

    # vue3 的响应式基础

    因为 vue2 使用的 Object.defineProperty,在面对特定类型时具有缺陷(看这篇)

    vue3 底层使用了 es6 中的 Proxy 来做数据劫持

    let obj = {
  name: "哈哈",
  age: 18,
}
let handlers = {
  get(target, key) {
    console.log(`get ${key}`)
    return target[key]
  },
  set(target, key, value) {
    console.log(`set ${key}:${value}`)
    target[key] = value
    return true
  },
}
let proxyObj = new Proxy(obj, handlers)

console.log(proxyObj.age)
proxyObj.name = "嘿嘿"
console.log(proxyObj.name)
proxyObj.height = 1.2
/**
output:
get age
18
set name:嘿嘿
get name
嘿嘿
set height:1.2
 */

    直接给对象新增属性也能监测到了

    vue3 的 reactive 函数,底层就是使用了 proxy,返回代理对象

    # reactive 的局限

    1. reactive 生成的响应式对象,如果使用了解构,会使解构出来的数据失去响应式
    const obj = reactive({ age: 19 })
let { age } = obj
obj.age = 20
console.log(age) // 19

    此外,reactive 对象的属性值传递给函数参数时,该值也失去响应式

    callSomeFunction(obj.age) // 传递的就是个普通值
    1. Proxy 只能代理对象类型,基本数据类型无法进行代理,vue 又提供了 ref 方法,用来包装基本数据类型

    RefImpl 伪代码

    class RefImpl {
  constructor(target) {
    let isObj = isObject(target)
    if (isObj) {
      this._value = toReactive(target)
    } else {
      this._value = target
    }
  }
  get value() {
    return this._value
  }

  set value(newVal) {
    let isObj = isObject(newValue)
    this._value = isObj ? toReactive(newVal) : newVal
    // ...触发响应式
  }
}

    使用 ref 会生成一个 ref 对象, 里面包含了 value 的 getter 和 setter,这也是为什么要用.value 解包 ref 值的原因

    包装对象类型时,底层会调 reactive 方法,将数据值包装为一个 proxy 对象,保存在_value 中

    包装基本数据类型时,不做处理只保存在_value中

    # 使用区别

    基本数据类型只能使用 ref 包装

    对象类型可以用 ref,也可以用 reactive,建议是用 ref,

    逻辑代码使用 ref 时,需要使用.value 解包数据

    ref 自动解包的情况

    • 顶层 ref 在模板中使用时 const age = ref(18)
    • ref 作为响应式对象的某个属性值时,在逻辑和模板中都会自动解包 const obj = reactive({ age: ref(18) })

    ref 作为响应式数组的元素或者作为普通对象的属性值时,在逻辑中都需要使用.value解包,在模板中,如果只读值,也有自动解包的效果,但是如果值参与其他运算,就得用.value解包

    const arr = reactive([ref(18)])
const obj2 = { age: ref(18) }

arr[0] += 1 //no
obj2.age += 1 //no
arr[0].value += 1 //ok
obj2.age.value += 1 //ok

    模板中

    <div>{{ arr[0] }} {{ obj2.age }}</div>
也能够正确的读值
<div>{{ arr[0] + 1 }} {{ obj2.age + 1 }}</div>
就不行了,因为 typeof arr[0], typeof obj2.age 其实是object,还是要用.value解包
