    # BFC

    # 是什么?

    html 中所有的元素,都运行在一种叫 formatting context(FC:格式化上下文)下,FC 又分为多个类型:

    inline formatting context

    block formatting context, 就是 BFC, 一个区域,作为 BFC 容器,其内部的元素有自己的布局规则,且布局规则不会影响到容器外部的其他元素

    但我依然无法明确自己的布局规则,到底说的啥玩意,网上的解答也没解开我的疑惑

    难道垂直方向上相邻元素的 margin 合并,就是规则么???

    # 如何开启一个bfc

    1. 文档元素 html 本身就已开启 bfc
    2. 使用了 float,值为 非none 的元素
    3. 使用了 position,且值为 absolute,fixed的元素
    4. display:inline-block 元素
    5. flex items, 前提是该子项没有再作为另一个 flex,grid,table 布局的容器
    6. 其他的,我没验

    使用了上面第 2、3条的元素,会隐式转为 inline-block 元素

    # 我理解的 bfc 特性

    # 1. margin 合并

    处于同一个BFC 容器下的相邻元素,他们之间的外边距会有一个合并效果,取两者间的大值

    <div class="container">
  <!-- 期望 块 1 和块 2 之间的间距为 100 px,但因为 bfc 特性,他们中间只有 50px -->
  <div class="a">1</div>
  <div class="b">2</div>
</div>

<style>
.container {
  border: 1px solid red;
  margin: 50px;
  display: inline-block;
}

.a {
  width: 50px;
  height: 50px;
  background-color: #888;
  margin: 50px;
}

.b {
  width: 50px;
  height: 50px;
  background-color: #aaa;
  margin: 50px;
}
</style>

    解决方法:

    1. 不解决, 开发中碰到这种情况,把真实间距(如 100px)设置在其中一个元素上
    2. 把其中一个元素用 bfc 块再包一层,但这样增加了元素,不好
