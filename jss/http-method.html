    # HTTP Method

    # get

    因为不能指定请求体, 请求头里也就没有 content-type(表示请求体的编码类型)和 content-length(请求体的长度)字段 参数以 a=b&c=d 形式拼接在 url?后面,参数要进行 urlencoded 编码,因为服务器以&作为分隔符来拆分参数,所以参数内容如有&+%等要编码后才能传递

    get 常用于获取数据,不用于动作类操作

    # post

    多用于增删改等动作类操作 content-type有三种

    • application/json

      请求体以 json 格式传递 常用于移动端与服务端交互

      {
  name:'haha',
  age:18
}

    • application/x-www-form-urlencoded

      文本类表单数据提交,参数和 get 请求的参数格式一致,只是作为请求体发送,不成为 url 后面拼接数据

      name=%E5%93%88%E5%93%88&age=18

    • multipart/form-data

      带有文件上传的表单,content-type 后面会拼接一个 boundary,作为分割参数的分隔符

      Content-Type:multipart/form-data; boundary=----WebKitFormBoundaryhWUszMbD5FLImHNb

      ------WebKitFormBoundaryhWUszMbD5FLImHNb
Content-Disposition: form-data; name="age"

18
------WebKitFormBoundaryhWUszMbD5FLImHNb
Content-Disposition: form-data; name="filename"; filename="deal_filename.py"
Content-Type: text/x-python-script


------WebKitFormBoundaryhWUszMbD5FLImHNb--

      最后一行后面多了个 -- 作为结束标志

    # content-type express配置

    使用 express 做后台开发时,需要中间件处理特定 content-type

    • body-parser 配置处理application/json 和 application/x-www-form-urlencoded的数据
    • multer 配置处理multipart/form-data 数据

    如果纯文本表单的 enctype 设为了multipart/form-data,那么也需要配置 multer 中间件

    multer 一般用作特定路由的中间件,而不作为全局的中间件,毕竟上传文件的接口还是少的
