    # JS任务

    js调度任务是单线程的,同一时刻只能做一件事,但是又会有些任务非常耗时,并不能去同步执行卡 UI,因此 js 里有个异步调度队列,碰见异步任务时,先放入队列中,待主线程的任务执行完毕后,再进行异步任务调度

    任务分为两种

    • 同步任务
    • 异步任务

    异步任务又分为宏任务,和微任务,分别放进宏任务队列,微任务队列

    宏任务:由宿主调度的任务,宿主一般指的是浏览器,和 node 环境 常见:setTimeout,setInterval

    微任务:由 js 调度的任务 常见:promise.then

    setTimeout(() => {
  console.log(1)
  }, 1000);

  new Promise((resolve, reject) => {
    console.log(2)
    setTimeout(() => {
      console.log(3)
      resolve()
    }, 5000);
  }).then(res => {
    console.log(5)
  })
  /**
   * output:
   * 2
   * 1
   * 3
   * 5
   * 
   * 2是同步,1,3 是宏任务,5 是微任务
   */
