    # 二. 重复请求的取消操作

    早期版本使用 CancelToken 取消,已过时

    现在使用 AbortController 来取消请求

    const requestRecords = {}
const controller = new AbortController()
service.interceptors.request.use(
  (config) => {

    /**
     * config 是这次请求的配置对象,包含了baseurl,url ,params, headers,data等
     * 根据 url+ params(get请求),url+data (post)请求,生成一个标识: urlmark
     * 拿到当前时间: time
     * 拿着 urlmark,去requestRecords找,
     * 如果有:说明之前执行过该请求,网慢等因素还没完成,
        获取 pretime = requestRecords[urlmark] 与 time 比较: time-pretime
        如果大于某个阈值,说明是用户重新请求了,更新最新的时间
          requestRecords:{
          urlmark: time
        }
        否则
          config.signal = controller.signal
          controller.abort()
     * 如果没有:直接记录
        requestRecords:{
          urlmark: time
        }
     */
    return config
  })

  service.interceptors.response.use(
  (response) => {
    /*
    response 是请求的响应数据,包含了 config,header,data(自己的服务器接口响应数据),status,statusText
    拿到 response.config,根据上面那个生成规则,得到 urlmark,然后从requestRecords移除即可
   */
  },(error)=>{
    /*
    error 是 AxiosError 类型,包含了 config,message,name等
    拿到 error.config, 根据上面那个生成规则,得到 urlmark,然后从requestRecords移除即可
    */
  })

    # 三. 响应拦截器

    有两个回调函数

    第一个是请求正常时调用,参数为这次请求的响应对象

    第二个是请求失败时调用,参数为 AxiosError对象

    常做的事情有

    • 隐藏 loading 框
    • 产生错误时,统一展示错误提示框(也可定制,headers.error传false进来,默认 true 统一处理)
    • 请求报错时: 向业务方回传 Promise.reject(error)
    • 请求正常
      • 如果是登录失效,过期等错误码,路由重定向至登录页面,清理本地数据
      • 接口数据不正常时:Promise.reject(response.data)回传数据
      • 数据正常时: 直接返回 response.data

    请求 headers 里面的 loading 和 error 为自定义字段

    # 四. 封装 axios 请求方法

    class Http {
  /* config:{
    headers,
    params,
    ...
  }
  */
  static get(url, params = {}, headers = {}) {
    return service.get(url, { params, headers })
  }
  static post(url, data = {}, headers = {}) {
    return service.post(url, data, { headers })
  }
  
  static postFile(url, file) {
    return service.post(url, file, { headers: { "Content-type": false } })
  }
  static getDown(url, params = {}) {
    return service.get(url, { params, responseType: "blob" })
  }
  static postDown(url, data = {}) {
    return service.post(url, data, { responseType: "blob" })
  }
}

export default Http
