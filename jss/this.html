# this指向问题

  • 顶层作用域下的普通函数,其内部的 this 指向 window, 严格模式下,指向 undefined
  • 使用 new 调用的函数内部,this 指向被构造出来的对象
  • 对象内部的方法,其 this 指向取决于调用方
  • 箭头函数里面的 this,来源于定义该箭头函数的上下文中的 this,且定义完以后就不再改变

修改 this 指向的几个方法

  • call, 修改 this 指向以后调用, 参数以可变参数形式传递
  • apply, 修改 this 指向以后调用, 参数以数组形式传递
  • bind, 修改 this 指向, 返回一个新的函数, 参数可以在调用时传递, 也可以在 bind 时传递
let obj = {
  name: '哈哈',
  print(a, b) {
    console.log(`${this.name}, ${a}, ${b}`)
  }
}
let obj2 = {
  name: '嘿嘿',
}

obj.print(1, 2)                 // 哈哈, 1, 2
obj.print.call(obj2, 3, 4)      // 嘿嘿, 3, 4
obj.print.apply(obj2, [5, 6])   // 嘿嘿, 5, 6

let print2 = obj.print.bind(obj2)
print2(7, 8)                    // 嘿嘿, 7, 8

let print3 = obj.print.bind(obj2, 9, 10)
print3()                        // 嘿嘿, 9, 10
