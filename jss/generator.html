    # 很绕的生成器

    # Generator 生成器函数

    普通函数的定义和调用

    function foo() {
  console.log('start')
  let a = 5
  let b = 6
  let c = a + b
  console.log('end')
  return c
}

let res = foo()
/** output:
 start
 end
 */

    生成器函数的定义和使用

    function* bar() {
  console.log('start')
  let a = 5
  yield a
  let b = 6
  yield b
  let c = a + b
  console.log('end')
  return c
}

let gen = bar()
/**
 * no outputs
 */

    定义时加了个*星号,这样这个函数就成了生成器函数,也叫状态机函数,里面定义了 n 个状态

    啥是状态机啊???

    啥是状态啊???

    状态: yield 或者 return 后面的东西, 叫状态

    这里有三个状态: 5、6、c(11)

    包含状态的函数,叫状态机,也就是这个 bar 生成器函数

    调用生成器函数,并不会执行该函数的函数体,而是返回一个状态机遍历对象 gen

    这个对象可以用来遍历状态...

    流程就是这么个流程

    # 使用 next 方法访问状态

    let n1 = gen.next() //{value: 5, done: false}
let n2 = gen.next() //{value: 6, done: false}
let n3 = gen.next() //{value: 11, done: true}

    每个状态由一个对象组成,有两个值,value 表示 yield 或者 return 后面的值, done 表示状态有没有遍历完成

    每调用一个 next方法,函数就执行一段代码,直到碰到一个 yield,

    所以 yield 起暂停函数的执行的作用, next 起恢复执行的作用

    # next方法的参数

    yield 表达式放在一个语句中时,要用()括起来,表示一个整体

    function* baz(x) {
  let y = 2 * (yield x * 5)
  let z = 100 + (yield y)

  return x + y + z
}

let gen = baz(5)
console.log(gen.next()) //{value: 25, done: false}
console.log(gen.next()) //{value: NaN, done: false}
console.log(gen.next()) //{value: NaN, done: true}

    yield 表达式,默认是没有返回值的,或者说返回值为 undefined,即

    let a = yield 2; //a === undefined

    上面 (yield x * 5) 值为 undefined, undefined * 2变成了 NaN

    (yield y) 值也为 undefined, undefined + 100 变成了 NaN

    注: next 方法获取的是 yield 后面的值,而不是 yield 表达式的值

    next方法用于修改上一个 yield 表达式的返回值,

    let gen = baz(5)
console.log(gen.next()) //{value: 25, done: false}
console.log(gen.next(4)) //{value: 8, done: false}
console.log(gen.next(8)) //{value: 121, done: true}

    调用第二个 next(4)的时候,4 会赋值给第一个(yield x * 5),这样 y 就变成了 2 * 4 = 8,也即第二个 next 得到的产出值

    调用第三个 next(8)的时候,8 会赋值给第二个(yield y),这样 z 就变成了 100 + 8 = 108

    x + y + z = 5 + 8 + 108 = 121,也即第三个 next 得到的产出值

    一般来说, 返回值都是获取的,但这里是修改的,和普通函数对比理解是有一点绕的

    # for...of

    for...of 可用于迭代支持 Iterator 协议的数据类型, 比如 Array, Map, Set, String, 生成器

    共同特征: 实现了Symbol.iterator接口

    Generator 函数执行后，返回一个迭代器对象。该对象本身也具有Symbol.iterator属性， 执行后返回自身。

    function* gen(){
  // some code
}

var g = gen();

g[Symbol.iterator]() === g
// true

function* foo() {
  yield 1;
  yield 2;
  yield 3;
  yield 4;
  yield 5;
  return 6;
}

for (let v of foo()) {
  console.log(v);
}
// 1 2 3 4 5
